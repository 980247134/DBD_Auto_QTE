# QTE Auto Tool v2.3

白圈 QTE 屏幕识别辅助工具，主要用于辅助黎明杀机（DBD）中的 QTE 场景。它会截取用户框选的 QTE 区域，识别红色指针和白色判定弧段，在指针进入判定区时自动按下 `Space`。

---

## 功能特点

### 白圈 QTE 专用

- 红色像素只用于识别指针。
- 白色像素只用于识别圆环和判定区。
- 不支持红圈 QTE。
- 不支持右键 M2 自动输入。
- 触发输入固定为 `Space`。

### 圆环遮罩

程序会在 MSS 截屏后立即应用圆形遮罩：

- 外圈遮罩：去除矩形截图四角干扰。
- 中心遮罩：去除圆心附近的 `Space/M2` 提示框干扰。
- 最终检测区域是一个可调节的圆环区域。

### 白色圆环完整性检查

为避免误判，程序会把遮罩后的圆环分为 8 个象限。

- 至少 7 个象限检测到白色像素，才认为 QTE 圆环存在。
- 圆环不完整时不会触发 `Space`。

### 指针判断点优化

红色指针判断点必须是真实红色像素。

当前逻辑会：

1. 识别红色指针方向。
2. 拟合红色指针中心线。
3. 在红色像素中寻找接近白圈半径、接近指针中心线的点。
4. 使用这个真实红色像素点作为命中判断点。

### 控制方式

- 点击 **启动检测** 开始
- 点击 **暂停 / 继续** 控制运行
- 点击 **停止** 结束检测
- 启动后有 2 秒倒计时，方便切回游戏窗口
- 运行时右上角显示状态：`QTE 运行` 或 `QTE 暂停`

### 配置保存

GUI 中提供 **保存当前配置** 按钮，可保存：

- 监控区域
- 检测频率
- 白色检测参数
- 红色检测参数
- 延迟补偿
- 外圈遮罩大小
- 中心遮罩大小

## 安装与启动

### 推荐方式：initialize.exe

双击：

```text
initialize.exe
```

它会自动：

1. 创建 `.venv` 虚拟环境。
2. 安装依赖。
3. 验证关键模块。
4. 生成 `run_qte.bat`。

初始化完成后，双击：

```text
run_qte.bat
```

`run_qte.bat` 会自动请求管理员权限启动程序。

### 手动方式

```bash
pip install -r requirements.txt
python main.py
```

## 使用步骤

### 1. 设置监控区域

点击 **全屏框选**，框住 QTE 圆环。

建议：

- 区域完整包含白色圆环。
- 尽量减少无关背景。
- 1080p 下推荐区域约 `160×160`。

当前 1080p 推荐区域：

```json
"region": {
  "left": 880,
  "top": 458,
  "width": 160,
  "height": 160
}
```

区域需要根据你的游戏分辨率、UI 缩放和窗口位置微调。

### 2. 启动检测

点击 **启动检测**。

程序会提示：

```text
请在2秒内切回游戏窗口
```

2 秒后开始检测，并在右上角显示运行状态。

### 3. 暂停 / 继续

点击 **暂停 / 继续**。

### 4. 停止

点击 **停止**。

## 1080p 推荐参数

以下参数来自当前实际调试配置，适合 **1080p / 白圈 QTE / 160×160 监控区域** 作为起点。

### 检测参数

| 参数 | 推荐值 | 说明 |
|---|---:|---|
| 检测频率 | `500Hz` | 屏幕检测循环频率，建议500Hz |
| 触发冷却 | `30ms` | 防止重复触发 |
| 预判补偿 | `0ms` | 默认关闭线性预判 |
| 白色亮度 | `166` | 白色圆环/判定区亮度阈值 |
| 色差容忍 | `50` | 白色 RGB 通道差异容忍 |

### 红色指针参数

| 参数 | 推荐值 | 说明 |
|---|---:|---|
| R 下限 | `180` | 红色通道最低值 |
| R 优势 | `50` | R 通道相对 G/B 的优势 |
| 红色亮度上限 | `220` | 过滤过亮区域 |
| 红色饱和度 | `0.4` | 红色饱和度阈值 |

### 遮罩参数

| 参数 | 推荐值 | 说明 |
|---|---:|---|
| 外圈大小 | `100%` | 缩小外圈，去掉矩形四角和外部背景 |
| 中心遮罩 | `0%` | 屏蔽中心提示框区域 |

### 触发参数

| 参数 | 推荐值 | 说明 |
|---|---:|---|
| 延迟补偿 | `3°` | 角度命中窗口偏移 |
| 输入方式 | `Space` | 当前版本只自动按 Space |

完整配置示例：

```json
{
  "version": "2.3",
  "region": {
    "left": 880,
    "top": 458,
    "width": 160,
    "height": 160
  },
  "params": {
    "target_hz": 500,
    "cooldown_ms": 30,
    "predict_ms": 0,
    "white_v_low": 166,
    "white_s_max": 50
  },
  "red_params": {
    "red_min_r": 180,
    "red_min_delta": 50,
    "red_max_avg": 220,
    "red_saturation": 0.4
  },
  "delay_degree": 3,
  "mask": {
    "outer_percent": 100,
    "center_percent": 0
  }
}
```

## 参数调试建议

### 检测不到白圈

可以尝试：

- 降低 **白色亮度**。
- 提高 **色差容忍**。
- 减小中心遮罩。
- 增大外圈大小。

### 检测不到红色指针

可以尝试：

- 降低 **R下限**。
- 降低 **R优势**。
- 降低或提高 **红色饱和度**，根据实际预览调节。

### 误触发

可以尝试：

- 提高 **白色亮度**。
- 降低 **色差容忍**。
- 增大中心遮罩。
- 降低外圈大小。

### 触发偏早或偏晚

调节 **延迟补偿**：

- 偏早：降低数值，例如 `3° → 0° → -3°`。
- 偏晚：提高数值，例如 `3° → 6° → 9°`。

### 游戏没有响应 Space

如果 GUI 中 **触发次数增加**，但游戏没有反应：

1. 确保游戏窗口在前台。
2. 使用 `run_qte.bat` 管理员权限启动。
3. 确认游戏 QTE 绑定包含 `Space`。
4. 避免独占全屏，建议无边框窗口或窗口化。

## 图像获取与输入方式

### 图像获取

项目支持两种采集方式：

#### MSS

默认方式，直接截取用户框选区域：

```text
mss.grab(region)
→ 圆形/中心遮罩
→ NumPy 阈值检测
→ 圆环角度分析
```

优点是延迟低、配置简单；缺点是少数游戏/显示模式可能阻止或影响截屏。

#### OBS 虚拟摄像头

当 MSS 截屏被游戏阻止时，可以切换到 OBS 采集：

```text
OBS 捕获游戏画面
→ 启动 OBS 虚拟摄像头
→ 工具读取虚拟摄像头画面
→ 按同一 region 坐标裁剪目标区域
→ 圆形/中心遮罩
→ NumPy 阈值检测
→ 圆环角度分析
```

OBS 使用步骤：

1. **配置OBS画布和虚拟摄像头**
   - 在 OBS 中添加游戏捕获、窗口捕获或显示器捕获
   - 设置画布分辨率为 1920×1080（设置 → 视频 → 基础画布分辨率）
   - **重要：设置虚拟摄像头输出分辨率为 1920×1080**
     - OBS 28+: 工具 → 虚拟摄像头 → 配置 → 输出分辨率
     - 如果找不到设置，请查看 `OBS配置指南.md`

2. **启动虚拟摄像头**
   - 在 OBS 中点击 **启动虚拟摄像头**

3. **在程序中配置**
   - 在工具 GUI 的 **图像采集方式** 中选择 `OBS 虚拟摄像头`
   - 点击 **🔍 扫描** 按钮查找可用摄像头
   - 根据扫描结果选择正确的 `OBS编号`（通常是 0 或 1）
   - 使用全屏框选或手动坐标设置 QTE 区域

4. **验证配置**
   - 启动检测后查看实时预览是否显示画面
   - 如果提示"监控区域超出画面范围"，说明虚拟摄像头输出分辨率不是 1920×1080
   - 详细解决方法请查看 `OBS配置指南.md`

注意：
- OBS 采集通常帧率低于 MSS，延迟也更高，但兼容性更好
- **OBS画布分辨率** 和 **虚拟摄像头输出分辨率** 是两个不同的设置，都需要设置为 1920×1080
- 如果 MSS 模式能正常工作，优先使用 MSS 模式

### 输入方式

项目支持两种输入方式：

1. **PyDirectInput 软件输入**
2. **Pico 2W USB 串口硬件输入**

默认通过 `pydirectinput` 发送：

```text
Space
```

如需使用 Pico 2W USB 串口，请先安装 `pyserial` 并参考 `docs/PICO_SETUP.md`。

## 注意事项

- 本工具不会读取游戏内存。
- 本工具不会注入游戏进程。
- 本工具依赖屏幕画面识别，亮度、分辨率、UI 缩放和窗口模式都会影响识别效果。
- 在带反作弊或联网环境中使用自动化工具可能违反游戏服务条款，请自行承担风险。
